### Определение MAC-адреса сетевого интерфейса

MAC-адрес сетевого интерфейса был определён из вывода команды ipconfig.  
Физический адрес беспроводного адаптера:

F8-FE-5E-6F-7B-EC

Данный адрес является уникальным идентификатором сетевого интерфейса на канальном уровне модели OSI.

### Анализ структуры MAC-адреса

MAC-адрес имеет длину 48 бит и представлен в шестнадцатеричном формате.  
Структура MAC-адреса F8-FE-5E-6F-7B-EC:

- первые 24 бита (F8-FE-5E) — OUI (Organizationally Unique Identifier), идентифицируют производителя сетевого оборудования (Intel);
- последние 24 бита (6F-7B-EC) — уникальный идентификатор конкретного сетевого интерфейса.

Анализ первого октета (F8 = 11111000₂):
- младший бит равен 0 — адрес индивидуальный (unicast);
- второй младший бит равен 0 — адрес глобально администрируемый.

Следовательно, MAC-адрес является индивидуальным и глобально назначенным производителем.

## Анализ протоколов транспортного уровня в Wireshark

### Захват трафика и анализ HTTP (TCP)

1. Запущена программа Wireshark и выбран активный сетевой интерфейс. После запуска был выполнен захват трафика.

2. В браузере выполнен переход на сайт, работающий по протоколу HTTP (например, info.cern.ch), после чего в Wireshark установлен фильтр http.

3. Проанализирован HTTP-запрос, переданный по протоколу TCP от устройства к веб-серверу.

   По данным захвата:
   - протокол канального уровня — Ethernet II;
   - IP-адрес источника — 192.168.1.69;
   - IP-адрес назначения — 188.184.67.127;
   - транспортный протокол — TCP;
   - порт источника — 52480 (эпhemeral/клиентский);
   - порт назначения — 80 (HTTP);
   - флаги TCP — PSH, ACK (передача данных в рамках установленного соединения);
   - длина TCP-сегмента (полезная нагрузка) — 448 байт;
   - длина TCP-заголовка — 20 байт (без опций);
   - окно приёма (Window) — 513; масштабирование окна — 256.

   Таким образом, HTTP-запрос передаётся поверх TCP, где соединение идентифицируется парой сокетов (IP:порт источника — IP:порт назначения), а параметры TCP (флаги, окна, номера последовательности/подтверждения) обеспечивают надёжную доставку данных.

![HTTP-запрос и параметры TCP (порт 80)](Screenshot_9.png){ #fig:010 width=80% }

4. Проанализирован HTTP-ответ сервера, полученный устройством по тому же TCP-соединению.

   По данным захвата:
   - IP-адрес источника — 188.184.67.127;
   - IP-адрес назначения — 192.168.1.69;
   - порт источника — 80 (HTTP);
   - порт назначения — 52480 (клиентский);
   - флаги TCP — PSH, ACK (сервер передаёт данные и подтверждает ранее принятые);

   Полученный ответ подтверждает двунаправленный обмен данными в рамках установленного TCP-сеанса: клиент отправляет запрос на порт 80 сервера, сервер отвечает на клиентский порт, назначенный ОС.

### Анализ этапов TCP handshake

Установление TCP-соединения происходит в три этапа (трёхстороннее рукопожатие).

**Первый этап — SYN**

   Клиент (192.168.1.69) инициирует соединение, отправляя серверу TCP-сегмент с установленным флагом SYN.

   Характеристики сегмента:
   - порт источника — 50071 (клиентский);
   - порт назначения — 80 (HTTP);
   - флаг TCP — SYN;
   - номер последовательности (Sequence Number) — 0;
   - размер окна — 64240;
   - длина TCP-заголовка — 32 байта (присутствуют TCP-опции, включая MSS и масштабирование окна).

   Данный сегмент сообщает серверу о намерении установить соединение и начальном номере последовательности клиента.

![TCP SYN от клиента](Screenshot_16.png){ #fig:017 width=80% }


**Второй этап — SYN, ACK**

   Сервер (188.184.67.127) отвечает клиенту сегментом с установленными флагами SYN и ACK.

   Характеристики сегмента:
   - порт источника — 80;
   - порт назначения — 50071;
   - флаги TCP — SYN, ACK;
   - номер последовательности — 0;
   - номер подтверждения — 1;
   - размер окна — 63744.

   ACK подтверждает получение SYN от клиента, а SYN со стороны сервера указывает его начальный номер последовательности.

**Третий этап — ACK**

   Клиент завершает установление соединения, отправляя серверу сегмент с установленным флагом ACK.

   Характеристики сегмента:
   - порт источника — 50071;
   - порт назначения — 80;
   - флаг TCP — ACK;
   - номер последовательности — 1;
   - номер подтверждения — 1.

   После передачи данного сегмента TCP-соединение считается установленным, и стороны могут начинать обмен прикладными данными.

![TCP ACK — завершение handshake](Screenshot_17.png){ #fig:018 width=80% }

7. После установления соединения были зафиксированы TCP-сегменты с флагами PSH, ACK, содержащие данные протокола HTTP, а также сегменты FIN, ACK, используемые для корректного завершения соединения.

8. Завершение TCP-сеанса подтверждается обменом сегментами с флагами FIN и ACK, после чего одна из сторон может отправить сегмент RST для принудительного закрытия соединения.

Проанализирован механизм установления TCP-соединения (three-way handshake) и подтверждена корректная работа сетевого стека операционной системы.
