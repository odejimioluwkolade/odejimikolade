# Контрольные вопросы

1. **Что такое DNS**  
   DNS (Domain Name System) — это система доменных имён, предназначенная для преобразования доменных имён в IP-адреса и обратно.

2. **Каково назначение кэширующего DNS-сервера**  
   Кэширующий DNS-сервер хранит результаты запросов к другим серверам, ускоряя последующие обращения к тем же доменам и снижая сетевую нагрузку.

3. **Чем отличается прямая DNS-зона от обратной**  
   Прямая зона преобразует доменные имена в IP-адреса, а обратная — наоборот, IP-адреса в доменные имена.

4. **В каких каталогах и файлах располагаются настройки DNS-сервера**  
   Основные файлы конфигурации находятся в каталогах `/etc/named.conf`, `/etc/named/`, `/var/named/`.  
   В них задаются параметры сервера, описания зон и пути к файлам с записями DNS.

5. **Что указывается в файле resolv.conf**  
   В файле `/etc/resolv.conf` указываются DNS-серверы, используемые системой для разрешения имён.

6. **Какие типы записи описания ресурсов есть в DNS**  
   Основные типы:  
   - **A** — IPv4-адрес узла;  
   - **AAAA** — IPv6-адрес;  
   - **NS** — сервер имён зоны;  
   - **SOA** — начальная запись зоны;  
   - **PTR** — обратная запись;  
   - **MX** — почтовый сервер;  
   - **CNAME** — псевдоним (алиас) имени.

7. **Для чего используется домен in-addr.arpa**  
   Этот домен применяется для организации обратного разрешения IP-адресов в доменные имена.

8. **Для чего нужен демон named**  
   Демон **named** — это служба, обеспечивающая работу DNS-сервера (Bind), обрабатывающая запросы клиентов и предоставляющая ответы.

9. **В чём заключаются основные функции slave-сервера и master-сервера**  
   Master-сервер хранит оригинальные файлы зон, а slave-сервер получает их копии и обслуживает клиентов при недоступности master.

10. **Какие параметры отвечают за время обновления зоны**  
    В записи **SOA** параметры `refresh`, `retry`, `expire` и `minimum` определяют интервалы обновления и срок действия зон.

11. **Как обеспечить защиту зоны от скачивания и просмотра**  
    Защиту можно реализовать с помощью ACL (Access Control List), директив `allow-transfer`, `allow-query`, а также DNSSEC.

12. **Какая запись RR применяется при создании почтовых серверов**  
    Для почтовых серверов используется запись **MX** (Mail Exchange).

13. **Как протестировать работу сервера доменных имён**  
    С помощью утилит **dig**, **host** или **nslookup**, выполняя запросы к доменам и проверяя корректность ответов.

14. **Как запустить, перезапустить или остановить службу**  
    Используются команды:  
    `systemctl start`, `systemctl restart`, `systemctl stop` и `systemctl status`.

15. **Как посмотреть отладочную информацию при запуске сервиса**  
    С помощью команды `journalctl -xeu <имя_сервиса>` или анализа логов в `/var/log/messages`.

16. **Где хранится отладочная информация по работе системы и служб**  
    Логи хранятся в `/var/log/` (например, `messages`, `secure`, `named.log`).  
    Просмотр осуществляется через `cat`, `less`, `tail -f` или `journalctl`.

17. **Как посмотреть, какие файлы использует процесс**  
    С помощью команды `lsof -p <PID>` или `lsof | grep <имя_процесса>`.  
    Пример: `lsof -p 1234` покажет все открытые файлы процесса.

18. **Примеры изменения сетевого соединения через nmcli**  
    ```
    nmcli connection modify eth0 ipv4.addresses 192.168.1.10/24  
    nmcli connection modify eth0 ipv4.gateway 192.168.1.1  
    nmcli connection modify eth0 ipv4.dns 127.0.0.1  
    nmcli connection up eth0
    ```

19. **Что такое SELinux**  
    SELinux (Security-Enhanced Linux) — подсистема безопасности ядра Linux, контролирующая доступ процессов к ресурсам.

20. **Что такое контекст SELinux**  
    Контекст — это метка безопасности, определяющая права доступа процессов и файлов в рамках политики SELinux.

21. **Как восстановить контекст SELinux после изменений**  
    Командой `restorecon -vR <каталог>` можно восстановить корректные метки для файлов.

22. **Как создать разрешающие правила политики SELinux**  
    С помощью утилиты `audit2allow`, анализирующей логи `/var/log/audit/audit.log` и формирующей необходимые правила.

23. **Что такое булевый переключатель в SELinux**  
    Булевый переключатель управляет отдельными аспектами политики безопасности, позволяя включать или отключать функции.

24. **Как посмотреть список переключателей SELinux**  
    Командой `getsebool -a` можно вывести все доступные переключатели и их состояние.

25. **Как изменить значение переключателя SELinux**  
    С помощью `setsebool <имя> on/off` или с параметром `-P` для сохранения изменений после перезагрузки.
