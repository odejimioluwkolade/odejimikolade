# Контрольные вопросы

1. **Вы хотите запретить удалённый доступ по SSH на сервер пользователю root и разрешить доступ пользователю alice. Как это сделать?**  
   Для этого в конфигурационном файле `/etc/ssh/sshd_config` необходимо запретить вход пользователю root, установив параметр `PermitRootLogin no`.  
   Дополнительно можно явно указать разрешённого пользователя, добавив директиву `AllowUsers alice`.  
   После внесения изменений требуется перезапустить службу SSH, чтобы новая конфигурация вступила в силу.

2. **Как настроить удалённый доступ по SSH через несколько портов? Для чего это может потребоваться?**  
   Для настройки доступа через несколько портов в файле `/etc/ssh/sshd_config` указываются несколько строк `Port` с нужными номерами портов.  
   Это может потребоваться для повышения отказоустойчивости, обхода сетевых ограничений или тестирования альтернативных конфигураций SSH без потери доступа к серверу при ошибках настройки.

3. **Какие параметры используются для создания туннеля SSH, когда команда ssh устанавливает фоновое соединение и не ожидает какой-либо конкретной команды?**  
   Для этого используются параметры `-f`, `-N` и `-L`.  
   Параметр `-f` переводит соединение в фоновый режим, `-N` указывает, что удалённая команда выполняться не будет, а `-L` задаёт правило локального перенаправления портов.

4. **Как настроить локальную переадресацию с локального порта 5555 на порт 80 сервера server2.example.com?**  
   Необходимо создать SSH-туннель с локальной переадресацией, указав локальный порт 5555 и целевой порт 80 сервера `server2.example.com`.  
   После установки такого соединения обращения к `localhost:5555` на клиентской машине будут перенаправляться на порт 80 удалённого сервера.

5. **Как настроить SELinux, чтобы позволить SSH связываться с портом 2022?**  
   Для этого порт 2022 необходимо добавить в контекст SELinux, предназначенный для SSH.  
   Это выполняется путём назначения порту 2022 типа `ssh_port_t`, после чего служба sshd получает разрешение на его использование.

6. **Как настроить межсетевой экран на сервере, чтобы разрешить входящие подключения по SSH через порт 2022?**  
   В настройках межсетевого экрана требуется разрешить входящие TCP-подключения на порт 2022.  
   Порт должен быть открыт как для текущей сессии, так и добавлен в постоянные правила, чтобы доступ сохранялся после перезагрузки сервера.
