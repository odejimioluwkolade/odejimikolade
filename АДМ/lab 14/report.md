# Цель работы

Приобретение навыков настройки доступа групп пользователей к общим ресурсам по протоколу SMB.

# Выполнение

## Настройка сервера Samba

1. На сервере установлены необходимые пакеты для работы Samba и клиентского доступа к разделяемым ресурсам. В процессе установки создана группа **sambagroup** с идентификатором GID 1010, пользователь **akabrikosov** добавлен в данную группу, а также создан каталог `/srv/sambashare`, предназначенный для размещения общего ресурса.

   ![Установка Samba и подготовка системы](Screenshot_1.png){ #fig:001 width=80% }

2. В конфигурационном файле `/etc/samba/smb.conf` добавлен раздел, описывающий общий ресурс **sambashare**. В настройках указан путь к каталогу `/srv/sambashare`, а также разрешена запись для пользователей, входящих в группу **sambagroup**.

   ![Настройка файла smb.conf](Screenshot_2.png){ #fig:002 width=80% }

3. Служба Samba запущена, проверен её текущий статус, подтверждающий готовность к обслуживанию подключений. После этого выполнена проверка наличия общего ресурса с использованием утилиты `smbclient`. Общий каталог **sambashare** успешно обнаружен.

   ![Проверка работы службы Samba](Screenshot_3.png){ #fig:003 width=80% }

4. Выполнен просмотр файла конфигурации межсетевого экрана `/usr/lib/firewalld/services/samba.xml`, в котором определены сетевые порты и параметры, используемые службой Samba.

   ![Просмотр файла samba.xml](Screenshot_4.png){ #fig:004 width=80% }

5. В межсетевом экране разрешена служба Samba для текущей и постоянной конфигурации. Дополнительно настроены права доступа к каталогу `/srv/sambashare`, а также выполнена настройка механизмов безопасности SELinux, включая изменение контекста безопасности и установку необходимых boolean-параметров.

   ![Настройка firewalld, прав доступа и SELinux](Screenshot_5.png){ #fig:005 width=80% }

6. Выполнена проверка контекста безопасности SELinux для каталога `/srv/sambashare`, которая подтвердила корректное применение типа `samba_share_t`.

   ![Проверка контекста SELinux](Screenshot_6.png){ #fig:006 width=80% }

7. Проверены идентификатор пользователя **akabrikosov** и его принадлежность к группам. Установлено, что пользователь успешно включён в группу **sambagroup**.

   ![Проверка UID и групп пользователя](Screenshot_7.png){ #fig:007 width=80% }

8. Под пользователем **akabrikosov** выполнена проверка возможности записи в общий каталог. В каталоге `/srv/sambashare` успешно создан тестовый файл, что подтверждает корректность настроек доступа.

   ![Создание файла в общем каталоге](Screenshot_8.png){ #fig:008 width=80% }

# Контрольные вопросы

1. Какова минимальная конфигурация для smb.conf для создания общего ресурса, который предоставляет доступ к каталогу /data?  
   Минимальная конфигурация включает глобальный раздел с указанием рабочей группы и раздел с описанием ресурса.  
   В разделе ресурса необходимо указать путь к каталогу `/data` и разрешить доступ к нему, задав параметры общего доступа.

2. Как настроить общий ресурс, который даёт доступ на запись всем пользователям, имеющим права на запись в файловой системе Linux?  
   Для этого в настройках ресурса Samba необходимо разрешить запись и не ограничивать доступ списками пользователей или групп.  
   Фактическая возможность записи будет определяться стандартными правами доступа файловой системы Linux.

3. Как ограничить доступ на запись к ресурсу только членам определённой группы?  
   Необходимо создать группу Linux, добавить в неё пользователей и указать эту группу в параметрах ресурса Samba, разрешив запись только для неё.  
   Дополнительно каталог должен принадлежать этой группе и иметь соответствующие групповые права.

4. Какой переключатель SELinux нужно использовать, чтобы позволить пользователям получать доступ к домашним каталогам на сервере через SMB?  
   Для этого используется boolean-переключатель `samba_enable_home_dirs`, который разрешает доступ к домашним каталогам пользователей через Samba.

5. Как ограничить доступ к определённому ресурсу только узлам из сети 192.168.10.0/24?  
   В настройках ресурса Samba необходимо указать разрешённую сеть с помощью параметров ограничения по IP-адресам, разрешив доступ только узлам из указанного диапазона.

6. Какую команду можно использовать, чтобы отобразить список всех пользователей Samba на сервере?  
   Для просмотра списка пользователей Samba используется команда управления базой пользователей Samba, позволяющая вывести все зарегистрированные учётные записи.

7. Что нужно сделать пользователю для доступа к ресурсу, который настроен как многопользовательский ресурс?  
   Пользователь должен существовать в системе Linux, быть добавлен в соответствующие группы и зарегистрирован в базе пользователей Samba.  
   Также ему необходимо знать свои учётные данные для аутентификации при подключении.

8. Как установить общий ресурс Samba в качестве многопользовательской учётной записи, где пользователь alice используется как минимальная учётная запись пользователя?  
   В настройках монтирования указывается минимальная учётная запись, от имени которой выполняется подключение, а реальные пользователи получают доступ в рамках многопользовательского режима.  
   При этом права доступа дополнительно контролируются файловой системой и группами.

9. Как можно запретить пользователям просматривать учётные данные монтирования Samba в файле /etc/fstab?  
   Для этого используется отдельный файл учётных данных с ограниченными правами доступа, который указывается в параметрах монтирования.  
   Сам файл должен быть доступен только администратору системы.

10. Какая команда позволяет перечислить все экспортируемые ресурсы Samba, доступные на определённом сервере?  
    Для просмотра списка экспортируемых ресурсов используется клиентская утилита Samba, позволяющая запросить у сервера перечень доступных общих каталогов.
