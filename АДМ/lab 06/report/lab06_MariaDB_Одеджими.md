# Контрольные вопросы

1. **Какая команда отвечает за настройки безопасности в MariaDB**  
   За выполнение начальной настройки безопасности отвечает команда `mysql_secure_installation`.  
   Она позволяет задать пароль root, удалить анонимных пользователей, отключить удалённый root-доступ и удалить тестовую базу.

2. **Как настроить MariaDB для доступа через сеть**  
   Доступ через сеть регулируется директивой `bind-address` в конфигурационных файлах MariaDB.  
   Чтобы разрешить сетевые подключения, параметр `bind-address` указывают на IP-адрес сервера или `0.0.0.0`.

3. **Какая команда позволяет получить обзор доступных баз данных после входа в MariaDB**  
   Для просмотра списка баз используется команда `SHOW DATABASES;`.

4. **Какая команда позволяет узнать, какие таблицы доступны в базе данных**  
   После выбора базы данных командой `USE`, список таблиц отображает команда `SHOW TABLES;`.

5. **Какая команда позволяет узнать, какие поля доступны в таблице**  
   Информация о структуре таблицы выводится командой `DESCRIBE имя_таблицы;`.

6. **Какая команда позволяет узнать, какие записи доступны в таблице**  
   Все записи таблицы отображает запрос `SELECT * FROM имя_таблицы;`.

7. **Как удалить запись из таблицы**  
   Удаление записи выполняется командой `DELETE` с указанием условия, например:  
   `DELETE FROM имя_таблицы WHERE поле = значение;`.

8. **Где расположены файлы конфигурации MariaDB и что можно настроить**  
   Основные конфигурационные файлы находятся в каталогах:  
   - `/etc/my.cnf`  
   - `/etc/my.cnf.d/`  
   Они позволяют настраивать: кодировки, сетевые параметры, пути данных, параметры кеширования, производительность и плагины.

9. **Где располагаются файлы с базами данных MariaDB**  
   По умолчанию файлы баз данных находятся в каталоге `/var/lib/mysql`.  
   Каждый каталог внутри соответствует базе данных MariaDB.

10. **Как сделать резервную копию базы данных и затем её восстановить**  
   Резервная копия создаётся утилитой `mysqldump`, а восстановление — через загрузку дампа в сервер MariaDB.  
   Примеры:  
   - резервная копия: `mysqldump база > файл.sql`  
   - восстановление: `mysql база < файл.sql`  
   Для сжатия дампа может использоваться `gzip` и `zcat`.
