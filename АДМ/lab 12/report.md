# Цель работы

Получение навыков по управлению системным временем и настройке синхронизации времени.

# Выполнение

## Настройка параметров времени и синхронизации

1. На сервере и клиенте выполнена проверка параметров настройки даты и времени с использованием команды timedatectl.  
   В результате установлено, что сервер и клиент находятся во временной зоне UTC (UTC, +0000).  
   Сетевая синхронизация времени включена, служба NTP активна, системные часы синхронизированы.

   ![Параметры даты и времени на сервере](Screenshot_1.png){ #fig:009 width=80% }

   ![Параметры даты и времени на клиенте](Screenshot_2.png){ #fig:010 width=80% }

2. На сервере и клиенте проверено текущее системное время с помощью команды date.  
   Вывод команды подтверждает корректную установку текущей даты и времени в соответствии с временной зоной UTC.

3. На сервере и клиенте выполнена проверка аппаратного времени с помощью команды hwclock.  
   Аппаратные часы синхронизированы с системным временем и также используют UTC.

## Управление синхронизацией времени (Chrony)

4. Проверены источники синхронизации времени на сервере и клиенте с использованием команды chronyc sources.  
   В выводе команды отображаются доступные серверы времени, их stratum, задержки и смещения.  
   Символ * обозначает текущий используемый источник, + — допустимый резервный, - — отклонённый.

5. На сервере открыт для редактирования файл /etc/chrony.conf, в который добавлена директива,
   разрешающая клиентам из локальной сети получать синхронизацию времени.

   ![Фрагмент chrony.conf на сервере](Screenshot_3.png){ #fig:015 width=80% }

6. После внесения изменений служба синхронизации времени chronyd на сервере была перезапущена.  
   Также выполнена настройка межсетевого экрана для разрешения сервиса NTP.

7. На клиенте в файле /etc/chrony.conf в качестве сервера времени указан server.akabrikosov.net.  
   Все остальные строки с директивой server были удалены.

   ![Настройка chrony.conf на клиенте](Screenshot_4.png){ #fig:016 width=80% }

8. После изменения конфигурации на клиенте выполнен перезапуск службы chronyd.  
   Повторная проверка источников времени показала, что клиент синхронизируется с сервером.

   ![Проверка источников времени после настройки](Screenshot_5.png){ #fig:017 width=80% }

10. В каталоге /vagrant/provision/server создан исполняемый скрипт ntp.sh, предназначенный для

    настройки межсетевого экрана и перезапуска службы.

    ![Скрипт ntp.sh для сервера](Screenshot_6.png){ #fig:018 width=80% }

11. Аналогичные действия выполнены на виртуальной машине client: создан каталог
    /vagrant/provision/client/ntp/etc, скопирован файл конфигурации chrony.conf и подготовлен
    исполняемый скрипт ntp.sh для применения настроек и перезапуска службы chronyd.

    ![Скрипт ntp.sh для клиента](Screenshot_7.png){ #fig:019 width=80% }

12. В конфигурационном файле Vagrantfile добавлены секции provision для виртуальных машин server и client,
    обеспечивающие автоматический запуск созданных скриптов при загрузке виртуальных машин.

# Контрольные вопросы

1. Почему важна точная синхронизация времени для служб баз данных  
   Точная синхронизация времени необходима для корректной фиксации транзакций,
   ведения журналов и репликации данных. Несогласованность времени может приводить
   к ошибкам при восстановлении данных, конфликтам транзакций и нарушению целостности
   журналов.

2. Почему служба проверки подлинности Kerberos сильно зависит от правильной синхронизации времени  
   Kerberos использует временные метки в билетах аутентификации для защиты от атак
   повторного воспроизведения. При значительном расхождении системного времени между
   клиентом и сервером билеты считаются недействительными, что делает аутентификацию
   невозможной.

3. Какая служба используется по умолчанию для синхронизации времени на RHEL 7  
   В RHEL 7 по умолчанию используется служба chrony, реализованная демоном chronyd.

4. Какова страта по умолчанию для локальных часов  
   Локальные часы имеют страту 10, что указывает на их использование только при
   отсутствии внешних источников синхронизации времени.

5. Какой порт брандмауэра должен быть открыт, если вы настраиваете свой сервер как одноранговый узел NTP  
   Для работы NTP необходимо открыть UDP-порт 123.

6. Какую строку нужно включить в конфигурационный файл chrony, если вы хотите быть сервером времени, даже если внешние серверы NTP недоступны  
   Для этого используется директива local с указанием страты, например:
   local stratum 10.

7. Какую страту имеет хост, если нет текущей синхронизации времени NTP  
   При отсутствии синхронизации с источниками NTP хост не имеет корректной страты
   и считается несинхронизированным, что отражается в выводе chrony как отсутствие
   выбранного источника времени.

8. Какую команду вы бы использовали на сервере с chrony, чтобы узнать, с какими серверами он синхронизируется  
   Для просмотра текущих источников синхронизации используется команда chronyc sources.

9. Как вы можете получить подробную статистику текущих настроек времени для процесса chrony вашего сервера  
   Подробную информацию о состоянии синхронизации, смещении и параметрах часов можно
   получить с помощью команды chronyc tracking.
